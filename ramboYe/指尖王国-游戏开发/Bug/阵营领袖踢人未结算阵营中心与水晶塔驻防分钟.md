# 阵营领袖踢人未结算阵营中心与水晶塔驻防分钟

> 关联版本：[[v1044]]

## 现象
- 期望：阵营领袖/管理员把驻防玩家踢出**阵营中心**或**水晶塔**时，被踢玩家应正常累计驻防时长（驻防分钟），用于驻防奖励领取。
- 实际：被踢出的玩家**没有累加阵营中心 / 水晶塔的驻防分钟**（进攻型驻防尤为明显，直接漏算）。

## 复现步骤
1. 冰髓争夺战争夺阶段，玩家驻防本阵营中心（进攻型/产冰霜怪）或水晶塔。
2. 阵营领袖/管理员点击踢出该玩家的驻防部队。
3. 查看被踢玩家的阵营中心 / 水晶塔驻防奖励进度（`DefendSeconds` / 驻防分钟）未增加。

## 根因分析

两个踢人 handler 在**召回部队之前**就提前清理了驻防记录，破坏了正常召回流程 `RecallTroop → giveUpDefend` 内部的结算判定。

### 阵营中心 `OnExpUnionCenterKickReq`（`game/wmap/internal/ctl_union_center.go:1435`）
原逻辑先 `center.RemoveDefypeID(req.TroopID)` 再 `RecallTroop(pt)`。

提前移除 defype 记录后，`TroopGiveUpDefendUnionCenter`（`ctl_union_center.go:673`）里：

```go
garrisonAtk := center.HasDefypeID(types.TroopDefypeFrostAtk, playerTroop.GetID()) // 恒为 false
...
if garrisonAtk && !wmap.playerHasAtkTroopsInBuild(center, pid) {
    // 进攻型驻防结算分支 —— 被跳过
    icm.CampCenterDefend.AddDefendSeconds(pid, seconds) // 驻防分钟，未执行
    wmap.settlePlayerAtkBuild(pid, expedition.DefendBBuildsScore, seconds)
}
```

因 `garrisonAtk` 恒为 `false`，进攻型（产冰霜怪）驻防的结算分支被跳过，`CampCenterDefend.AddDefendSeconds`（驻防分钟）不再执行，且错误落入防御分支用错了时间记录。

> 阵营中心的驻防分钟（`CampCenterDefendMgr.DefendSeconds`，见 `game/wmap/internal/expedition/camp_center_defend_mgr.go:9`）按设计只统计**进攻型驻防**，恰好就是被该 bug 跳过的分支。

### 水晶塔 `OnExpExpCrystalTowerKickReq`（`game/wmap/internal/ctl_crystal_tower.go:726`）
原逻辑显式 `removeCrystalTowerDefender(tower, pt)` 后又 `RecallTroop(pt)`，而 `RecallTroop` 内部会再次经 `giveUpDefend → TroopGiveUpDefendCrystalTower → removeCrystalTowerDefender`，属于**重复移除/结算**。当前仅靠 `util.TimeFromNow(0) == 0` 才避免重复计数（`vendor/.../util/time.go:98`），逻辑脆弱。

## 修复方案

统一改为**只走 `RecallTroop` 这条与玩家主动放弃驻防一致的规范结算路径**，提前清理仅作为部队已不存在时的兜底。

- 涉及文件：
  - `game/wmap/internal/ctl_union_center.go`（`OnExpUnionCenterKickReq`）
  - `game/wmap/internal/ctl_crystal_tower.go`（`OnExpExpCrystalTowerKickReq`）

### 阵营中心
- 仅当 `pt == nil`（异常编队/部队已不存在）时才 `center.RemoveDefypeID` 兜底清理。
- 部队存在时直接 `RecallTroop(pt)`，由 `giveUpDefend → TroopGiveUpDefendUnionCenter` 正常结算（含进攻/防御两种驻防），再 `BroadcastUnitNtfImpl(center)`。

### 水晶塔
- 去掉显式 `removeCrystalTowerDefender` 调用，只保留 `RecallTroop(pt)`，由统一路径结算驻防分钟，消除重复移除。

### 附带
- 两个 handler 各加一条英文 info 日志，打印发起者与被踢玩家：
  `operatorPlayerID:%v kickedPlayerID:%v`。
- 另在 `OnUnionWarKickReq` 的 `executeKickTroop`（`game/wmap/internal/ctl_union_war.go`）也补了同风格踢人日志。
