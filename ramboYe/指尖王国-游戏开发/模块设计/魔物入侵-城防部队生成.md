# 魔物入侵 - 城防部队生成

## 目标
梳理「魔物（怪物 NPC）入侵攻击玩家城堡时，城堡侧生成的防御部队是什么类型」这一问题。

**结论**：生成的防御部队是 **集结部队（`RallyTroop`）**，项目里称为「城防部队」；其成员是用城内玩家自己兵力构造的 `PlayerTroop`（临时编队），战斗类型为 `BattleType_City`（城防战）。

> 印证注释（`game/wmap/internal/ctl_battle_settle.go:115`）：
> `// 魔物入侵 城防部队 rally_troop -> 魔物怪 npc_troop`
> 即魔物为 `npc_troop`，城堡侧应战的是 `rally_troop`（城防集结部队）。

## 数据结构 / 类型
- **怪物（攻击方）**：`npc_troop`（NPC 部队），魔物入侵类别为 `gdconf.MonsterType_Demon_Monster`。
- **城防部队（防守方）**：`mtroop.RallyTroop`（集结部队）。
  - 成员单元：`mtroop.PlayerTroop`（玩家部队），第一个为首领，其余为跟随者。
  - 成员标记：`pt.IsTempLineUp = true`（临时编队）、`pt.SetIsDefend(true)`（防守状态）。
  - 集结体标记：`rt.SetIsDefender(true)`、`rt.SetDefendCtx(city.GetType(), city.GetID())`、`rt.SetCityID(...)`。
  - 战斗类型：`gdconf.BattleType_City`。

## 核心流程
1. NPC 部队到达玩家城堡 → `WMap.NtroopAtkPlayerCityArrived`（`game/wmap/internal/ctl_march_event.go:817`）。
2. 先查是否已有城防部队 `getPlayerCityDefender`（`ctl_playertroop.go:1403`，按 `city.GetDefTroopID()` 取 `RallyTroop`）：
   - **已存在且在战斗中** → 直接 `troopAtkTroop(npcTroop, defender)`，无需新建（`ctl_march_event.go:832`）。
   - **不存在** → 异步加载被攻击方玩家数据 `M2PCityBeAttackReq`，回调 `pTroopAtkMapCityLoadPlayerCb`。
3. 回调 `pTroopAtkMapCityLoadPlayerCb`（`ctl_playertroop.go:946`）：
   - 校验部队/玩家/城堡是否存在，失败重试一次（`times < 1`）。
   - 加载成功 → `updatePlayerCityDefender(...)` → `newPlayerCityDefender(...)` 生成城防部队。
   - `wmap.troopAtkTroop(troop, defender)` 开战。
4. 生成逻辑 `newPlayerCityDefender`（`ctl_playertroop.go:1449`）：
   - 用 `ack` 返回的兵力/英雄/属性逐个构造 `PlayerTroop`（标记临时编队 + 防守）。
   - 若已有 `oldDefender` → 合并进旧集结部队（`InsertBefore` / `AddTroop`），返回旧集结部队。
   - 若无 → `mtroop.NewRallyTroop(rtId, battleMgr, t, tops...)` 新建集结部队，设置防守上下文、城市半径，`AddUnit` 入图，`city.SetDefTroopID(rt.GetID())`，战斗类型设为 `BattleType_City`。

## 核心代码片段
```go
// game/wmap/internal/ctl_playertroop.go:1568
// 将所有的城防部队转化为集结部队
rt := mtroop.NewRallyTroop(rtId, battleMgr, t, tops...)
rt.SetStartUnitCtx(&munit.StartUnitCtx{ID: city.GetID()})
rt.SetDefendCtx(city.GetType(), city.GetID())
rt.SetCityID(city.GetID())
rt.SetIsDefender(true)
...
wmap.SetBattleTypeByMap(rt, gdconf.BattleType_City)
```

## 涉及协议 / 配置
- proto / iproto：`M2PCityBeAttackReq`、`P2MCityBeAttackAck`、`M2PCreateCityDefenderFailedNtf`。
- 战斗类型：`gdconf.BattleType_City`。
- 怪物类别：`gdconf.MonsterType_Demon_Monster`。

## 风险与边界
- **多编队同时到达**：用 `city.GetDefTroopID()` 查已有城防，若已在战斗则直接攻击已有集结部队，避免重复创建（`ctl_playertroop.go:990`）。
- **加载失败**：仅重试一次（`times < 1`）；魔物类别失败时走 `DevilAttackMgr` 失败结算，必要时检测最后一波是否胜利。
- 城防部队成员为临时编队（`IsTempLineUp`），区别于玩家手动派出的常驻部队。
