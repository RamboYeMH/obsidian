# 第12章-反射 面试题

关联：[[第12章-反射]]、[[00-Go语言学习索引]]

## Q1：反射解决什么问题？

**答：** 在运行时检查和操作未知类型的值，例如编码解码、配置绑定、ORM、通用格式化。

## Q2：`reflect.TypeOf` 和 `reflect.ValueOf` 分别返回什么？

**答：** `TypeOf` 返回动态类型，`ValueOf` 返回动态值的反射表示。

## Q3：Type 和 Kind 有什么区别？

**答：** Type 是具体命名类型，Kind 是底层类别。例如 `type Age int` 的 Type 是 `Age`，Kind 是 `int`。

## Q4：为什么 `reflect.ValueOf(x).SetInt(...)` 可能 panic？

**答：** 因为该 Value 不可设置。要修改变量，通常需要传指针：`reflect.ValueOf(&x).Elem()`。

## Q5：什么是可寻址？

**答：** 一个值有稳定地址，可以通过地址修改。反射修改通常要求 Value 可寻址且可设置。

## Q6：struct tag 是什么？

**答：** struct 字段后的原始字符串元数据，常被 JSON、ORM、表单库通过反射读取。

## Q7：反射有什么缺点？

**答：** 编译期类型检查变弱，运行时 panic 风险增加，代码可读性下降，性能通常更差。

## Q8：什么时候不该用反射？

**答：** 能用接口、泛型、普通函数或代码生成清晰解决时，不应优先用反射。

## Q9：反射能调用方法吗？

**答：** 可以通过 `Value.MethodByName` 或 `Type` 的方法集查询，但要处理参数、返回值和导出限制。

## Q10：JSON 为什么会用到反射？

**答：** JSON 编码器需要在运行时遍历任意 struct 字段、读取 tag，并按字段类型编码或解码。
