## 知识点

### 1. 命名规则和导出规则
Go 标识符必须以 Unicode 字母或 `_` 开头，后面可以跟字母、数字或 `_`。大小写敏感，`heapSort` 和 `Heapsort` 是两个不同名字。

最重要的规则：包级名字以大写字母开头就是导出的，可以被其他包访问；小写开头只在当前包内部可见。包名通常使用小写。

命名风格：Go 更偏短命名，作用域越小名字越短；作用域越大、语义越重要，名字应更清楚。多词组合优先用驼峰，例如 `parseRequestLine`，不要写 `parse_request_line`。

### 2. 四类声明
Go 主要有四类声明：

```go
var   // 变量
const // 常量
type  // 类型
func  // 函数
```

源文件结构通常是：

```go
package xxx

import (...)

var/const/type/func declarations
```

包级声明在整个包内可见，不局限于当前文件。函数内部声明只在局部作用域内可见。示例：`vendor/gopl.io/ch2/boiling/main.go:12`、`vendor/gopl.io/ch2/ftoc/main.go:18`。

### 3. 变量和零值
变量声明形式：

```go
var name type = expression
```

类型和初始化表达式可以省略其中一个。没有显式初始化时，Go 会使用零值：

| 类型 | 零值 |
|---|---|
| 数值 | `0` |
| bool | `false` |
| string | `""` |
| 指针、slice、map、chan、函数、接口 | `nil` |
| 数组、结构体 | 每个元素或字段都是对应类型零值 |

Go 不存在未初始化变量。写 Go 时应尽量让自定义类型的零值也有意义。

### 4. 短变量声明 `:=`
`:=` 只能在函数内部使用，用于声明并初始化局部变量：

```go
f, err := os.Open(name)
```

必须牢记：`:=` 是声明，`=` 是赋值。

短声明左边至少要有一个新变量。如果同一词法域中已有变量，`:=` 会对已有变量赋值，同时声明新变量：

```go
in, err := os.Open(infile)
out, err := os.Create(outfile) // out 是新变量，err 是赋值
```

但如果外层已有变量，内层使用 `:=` 可能会重新声明并遮蔽外层变量，这是第二章最容易出 bug 的点。

### 5. 指针
`&x` 取变量地址，类型是 `*T`；`*p` 访问指针指向的变量。

```go
x := 1
p := &x
fmt.Println(*p) // 1
*p = 2
fmt.Println(x)  // 2
```

Go 支持指针，但不支持 C 那种指针运算。函数可以安全返回局部变量地址，因为 Go 编译器会根据逃逸分析决定变量放在栈上还是堆上。

`flag` 包的示例很适合理解指针：`flag.Bool`、`flag.String` 返回的是指针，使用值时要写 `*n`、`*sep`。示例：`vendor/gopl.io/ch2/echo4/main.go:16`。

### 6. `new` 函数
`new(T)` 创建一个 T 类型匿名变量，初始化为 T 的零值，并返回 `*T`。

```go
p := new(int)
fmt.Println(*p) // 0
*p = 2
```

`new` 不是关键字，只是预定义函数。实际项目中，结构体更多使用字面量创建，`new` 使用频率并不高。

### 7. 生命周期和逃逸
作用域是编译期概念，描述名字在源代码哪里可见；生命周期是运行期概念，描述变量在内存中存在多久。

局部变量不一定只活到函数返回。如果局部变量地址被外部引用，它会逃逸，函数返回后仍然有效。示例来源：`ch2/ch2-03.md:290`。

重点理解：变量放栈还是堆，不由 `var` 或 `new` 决定，而由编译器根据逃逸分析决定。

### 8. 赋值和元组赋值
赋值可以更新普通变量、指针指向的变量、结构体字段、数组/slice/map 元素。

元组赋值会先计算右边所有表达式，再统一赋给左边变量，所以可以优雅交换变量：

```go
x, y = y, x
```

GCD 和 Fibonacci 都能用元组赋值写得很简洁：`ch2/ch2-04.md:41`、`ch2/ch2-04.md:52`。

多个返回值也常配合元组赋值：

```go
f, err = os.Open("foo.txt")
v, ok = m[key]
v, ok = x.(T)
v, ok = <-ch
```

### 9. 可赋值性
无论显式赋值还是隐式赋值，右边的值必须能赋给左边变量。当前阶段可以先记住：普通变量赋值要求类型匹配，`nil` 可以赋给指针或引用类型。

函数调用传参、函数返回、复合字面量初始化，本质上也都有隐式赋值行为。

### 10. 命名类型和类型转换
类型声明创建一个新的命名类型：

```go
type Celsius float64
type Fahrenheit float64
```

即使 `Celsius` 和 `Fahrenheit` 底层类型都是 `float64`，它们也是不同类型，不能直接混用。这能避免把不同业务含义的数据误加误比。

类型转换写作 `T(x)`。注意：`Celsius(f)` 是类型转换，不是温度换算；`FToC(f)` 才是业务换算函数。

示例：`vendor/gopl.io/ch2/tempconv/tempconv.go:11`、`vendor/gopl.io/ch2/tempconv/conv.go:11`。

### 11. 方法预告：`String()`
命名类型可以定义方法：

```go
func (c Celsius) String() string { return fmt.Sprintf("%g°C", c) }
```

当使用 `fmt.Println(c)` 或 `%v`、`%s` 打印时，`fmt` 会优先使用 `String()`。示例：`vendor/gopl.io/ch2/tempconv/tempconv.go:20`。

### 12. 包和文件
一个包可以由多个 `.go` 文件组成。同一个包内，不同文件的包级声明互相可见；但每个文件的 `import` 只对当前文件有效。

包用于模块化、封装、单独编译和代码复用。包内大写开头的包级名字导出，小写开头的包级名字隐藏实现细节。

导入路径是全局唯一字符串，例如：

```go
import "gopl.io/ch2/tempconv"
```

包名通常是导入路径最后一段，例如 `tempconv`。

### 13. 包初始化和 `init`
包初始化顺序：

1. 先初始化被依赖的包。
2. 每个包只初始化一次。
3. 包级变量按依赖和声明顺序初始化。
4. 每个文件中的 `init` 按声明顺序自动执行。
5. `main` 包最后初始化，然后执行 `main.main()`。

`init` 不能被手动调用或引用。适合做复杂表格预计算、注册等初始化工作。示例：`vendor/gopl.io/ch2/popcount/main.go:13`。

### 14. 作用域和遮蔽
作用域是名字在源代码中的可见范围。查找名字时，编译器从最内层词法块向外找。

内部声明会遮蔽外部同名声明：

```go
var cwd string

func init() {
    cwd, err := os.Getwd() // 错误倾向：这里声明了局部 cwd
    _ = err
    _ = cwd
}
```

正确写法是提前声明 `err`，然后使用赋值：

```go
var cwd string

func init() {
    var err error
    cwd, err = os.Getwd()
    if err != nil {
        log.Fatalf("os.Getwd failed: %v", err)
    }
}
```

这是第二章最重要的实战坑。来源：`ch2/ch2-07.md:118`。

## 重点关注点
- 大写开头导出，小写开头包内可见，这既是命名规则，也是封装机制。
- `:=` 只能用于函数内，并且可能引入变量遮蔽；写 `err` 处理代码时尤其要小心。
- 零值是 Go 的重要设计，尽量让类型的零值可用。
- `new(T)` 返回 `*T`，但变量是否上堆不由 `new` 决定，而由逃逸分析决定。
- 指针能修改外部变量，也会产生别名；别名越多，状态越难追踪。
- 元组赋值先计算右边，再更新左边，适合交换变量和状态推进。
- 命名类型即使底层类型相同，也不是同一类型；需要显式转换。
- 类型转换和业务换算不是一回事，例如 `Celsius(f)` 不等于 `FToC(f)`。
- 每个文件必须自己导入需要的包，不能依赖同包其他文件的 import。
- 包初始化发生在 `main` 前，`init` 自动执行，但不要滥用。
- 作用域是编译期文本范围，生命周期是运行期存在时间，不要混淆。
