# 第7章-接口 面试题

关联：[[第7章-接口]]、[[00-Go语言学习索引]]

## Q1：Go 接口和 Java interface 最大区别是什么？

**答：** Go 是隐式实现。类型只要拥有接口要求的方法集合，就自动实现接口，不需要显式声明 `implements`。

## Q2：接口值由什么组成？

**答：** 接口值由动态类型和动态值组成。只有动态类型和动态值都为空时，接口值才等于 nil。

## Q3：为什么“nil 指针赋给接口后，接口不等于 nil”？

**答：** 因为接口已经记录了动态类型。例如 `var b *bytes.Buffer = nil; var w io.Writer = b`，此时 `w` 的动态类型是 `*bytes.Buffer`，动态值是 nil，所以 `w != nil`。

## Q4：`T` 和 `*T` 实现接口的条件有什么区别？

**答：** `T` 的方法集只包含值接收器方法；`*T` 的方法集包含值接收器和指针接收器方法。如果接口要求的方法定义在指针接收器上，通常只有 `*T` 实现该接口。

## Q5：什么是类型断言？

**答：** 类型断言从接口值中检查或提取具体类型：`v, ok := x.(T)`。带 `ok` 的形式失败时不会 panic。

## Q6：type switch 适合什么场景？

**答：** 适合对接口值按动态类型分支处理，常用于格式化、解码、错误分类等。

## Q7：为什么推荐小接口？

**答：** 小接口更容易被已有类型实现，也更容易测试和组合。`io.Reader`、`io.Writer` 就是典型小接口。

## Q8：接口应该由提供方定义还是调用方定义？

**答：** Go 中通常由调用方按需要定义小接口。提供方返回具体类型，避免提前抽象。

## Q9：`error` 为什么是接口？

**答：** `error` 只有 `Error() string` 方法，任何类型只要实现它都能作为错误返回，同时还可以携带额外上下文供类型断言使用。

## Q10：`sort.Interface` 的三个方法是什么？

**答：** `Len() int`、`Less(i, j int) bool`、`Swap(i, j int)`。
