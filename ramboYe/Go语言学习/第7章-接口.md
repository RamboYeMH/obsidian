## 知识点

### 1. 接口是合约

接口只描述方法，不描述字段和实现。一个类型只要拥有接口要求的全部方法，就自动实现该接口，不需要显式声明。

```go
type Writer interface {
    Write(p []byte) (n int, err error)
}
```

### 2. 接口类型和接口组合

接口的方法集合可以通过嵌入其他接口组合出来：

```go
type ReadWriter interface {
    Reader
    Writer
}
```

接口越小越容易复用。`io.Reader`、`io.Writer` 是最典型的例子。

### 3. 实现接口的条件

实现关系由方法集决定。值类型 `T` 的方法集只包含值接收器方法；指针类型 `*T` 的方法集包含值接收器和指针接收器方法。这个规则会影响 `T` 和 `*T` 谁能赋给接口变量。

### 4. 接口值

接口值由两部分组成：动态类型和动态值。只有两者都为空时，接口值才等于 nil。

```go
var w io.Writer
w == nil // true

var buf *bytes.Buffer
w = buf
w == nil // false，动态类型是 *bytes.Buffer，动态值是 nil
```

这是 Go 面试和线上 bug 中非常常见的点。

### 5. 标准库接口

- `sort.Interface`：`Len`、`Less`、`Swap`，用于自定义排序。
- `http.Handler`：`ServeHTTP`，用于 HTTP 请求分发。
- `error`：`Error() string`，错误值的统一抽象。

这些接口都很小，但组合能力强。

### 6. 类型断言和类型分支

类型断言用于从接口值中取出具体类型或判断是否实现更具体的接口：

```go
if rw, ok := w.(io.ReadWriter); ok {
    // rw 同时支持读写
}
```

type switch 适合按动态类型分支处理：

```go
switch x := x.(type) {
case nil:
case int:
case string:
default:
}
```

### 7. 接口设计原则

不要为了“将来可能用到”提前抽象接口。Go 更常见的做法是由调用方定义小接口，生产方只提供具体类型。

## 重点关注点

- 接口是隐式实现，不需要 `implements`。
- 接口值包含动态类型和动态值。
- “包含 nil 指针的接口”不是 nil 接口。
- 方法集决定 `T` 或 `*T` 是否实现接口。
- 优先设计小接口，让调用方按需求定义接口。
