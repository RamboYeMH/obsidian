## 知识点

### 1. 函数签名

函数类型由参数列表和返回值列表决定，变量名不影响签名。两个函数形参和返回值类型一一对应即为相同签名：

```go
func add(x int, y int) int { return x + y }
fmt.Printf("%T\n", add) // "func(int, int) int"
```

Go 没有默认参数值，也不能通过参数名传参。所有参数必须按顺序提供。`ch5/ch5-01.md:14`

### 2. 命名返回值与 bare return

返回值可以命名，命名后它们是函数最外层的局部变量，初始化为零值。所有返回值都有名字时，`return` 可以省略操作数（bare return）：

```go
func CountWordsAndImages(url string) (words, images int, err error) {
    resp, err := http.Get(url)
    if err != nil {
        return // 等价于 return 0, 0, err
    }
    // ...
    return // 等价于 return words, images, nil
}
```

bare return 减少重复但降低可读性，多处 return 时慎用。`ch5/ch5-03.md:48`

### 3. 值传递

Go 所有参数都是值传递。引用类型（指针、slice、map、channel、函数）传递的是引用的副本，通过副本仍可修改被指向的内容。`ch5/ch5-01.md:51`

### 4. Go 使用可变栈

大多数语言函数调用栈大小固定（64KB–2MB），递归深时会溢出。Go 的 goroutine 使用可变栈，按需增长，初始很小。因此 Go 递归不需要担心栈溢出。`ch5/ch5-02.md`

### 5. 多返回值惯用法

Go 函数常见模式是返回 `(结果, error)`，调用者必须显式处理：

```go
links, err := findLinks(url)
if err != nil {
    // 处理错误
}
```

不关心某个返回值时用 blank identifier `_`。可以将多返回值函数的调用直接作为另一个多参数函数的参数：

```go
log.Println(findLinks(url)) // 等价于 links, err := findLinks(url); log.Println(links, err)
```

`ch5/ch5-03.md:30`

### 6. 错误处理五种策略

| 策略 | 适用场景 | 示例 |
|---|---|---|
| 传播错误 | 子调用失败 → 函数失败 | `return nil, err` |
| 重试 | 偶发、不可预知错误 | 指数退避重试，设超时 |
| 输出并退出 | 不可恢复，main 中使用 | `log.Fatalf(...)` |
| 只记录日志 | 程序可继续运行 | `log.Printf(...)` |
| 忽略 | 明确不影响逻辑 | 显式注释说明意图 |

错误信息不要大写开头，不要换行符，便于 grep。构造错误链时用 `fmt.Errorf` 加上下文前缀。`ch5/ch5-04.md`

### 7. io.EOF

`io.EOF` 是 `io` 包定义的哨兵错误值，表示文件/流读取完毕。用 `==` 比较：

```go
if err == io.EOF {
    break
}
```

`ch5/ch5-04.md:（EOF部分）`

### 8. 函数是一等值

函数可以赋值给变量、作为参数传递、作为返回值返回。函数类型的零值是 nil，调用 nil 函数值会 panic。函数值不可比较，不能作为 map key。`ch5/ch5-05.md`

```go
f := square       // 赋值
f(3)              // 调用
var f func(int) int  // nil
if f != nil { f(3) }
```

### 9. 闭包

匿名函数可以捕获外层函数的变量，形成闭包。闭包持有的是变量的引用，而不是值的副本：

```go
func squares() func() int {
    var x int
    return func() int {
        x++
        return x * x
    }
}
f := squares()
f() // 1, f() // 4, f() // 9 — x 的状态在调用间持续
```

变量生命周期不由作用域决定，`squares` 返回后 `x` 仍存活于闭包中。`ch5/ch5-06.md:17`

### 10. 循环变量陷阱（重要）

在循环中将循环变量捕获进闭包时，所有闭包共享同一个变量地址，循环结束后变量值是最后一次迭代的值：

```go
// 错误写法
for _, dir := range tempDirs() {
    rmdirs = append(rmdirs, func() {
        os.RemoveAll(dir) // 所有闭包共享同一个 dir，执行时都是最后一个值
    })
}

// 正确写法：在循环内创建副本
for _, dir := range tempDirs() {
    dir := dir // 创建新的局部变量
    rmdirs = append(rmdirs, func() {
        os.RemoveAll(dir)
    })
}
```

同样的问题出现在 `go` 语句和 `defer` 语句中。`ch5/ch5-06.md:252`

### 11. 可变参数

在最后一个参数类型前加 `...`，函数体内该参数表现为 slice：

```go
func sum(vals ...int) int { ... }
sum(1, 2, 3)
values := []int{1, 2, 3}
sum(values...) // 传 slice 时加 ...
```

`func f(...int)` 和 `func g([]int)` 是不同类型。`ch5/ch5-07.md:8`

### 12. defer 执行时机

`defer` 语句在函数执行完毕时（无论 return 还是 panic）才执行，多个 defer 按**逆序**执行。参数表达式在 defer 语句处立即求值，函数体延迟执行：

```go
defer resp.Body.Close()   // 获取资源后立即 defer，确保所有路径都释放
defer mu.Unlock()         // 加锁后立即 defer
defer trace("foo")()      // 注意双括号：trace() 立即执行，返回的函数延迟执行
```

`ch5/ch5-08.md`

### 13. defer 观察/修改返回值

defer 中的匿名函数可以读写命名返回值：

```go
func triple(x int) (result int) {
    defer func() { result += x }()
    return double(x) // result = 2x，defer 后 result = 3x
}
```

`ch5/ch5-08.md`

### 14. 循环中的 defer 陷阱

循环体内的 defer 直到函数返回才执行，不是每次迭代执行，可能耗尽文件描述符：

```go
// 危险：所有 f.Close() 在函数末尾才执行
for _, filename := range filenames {
    f, _ := os.Open(filename)
    defer f.Close()
}

// 正确：封装成函数，每次调用完毕即关闭
for _, filename := range filenames {
    doFile(filename)
}
```

`ch5/ch5-08.md`

### 15. panic

`panic` 用于不可恢复的严重错误（程序内部逻辑不一致、bug）。panic 发生后：
1. 当前函数停止运行
2. 当前 goroutine 的 defer 函数按逆序执行
3. 程序崩溃并输出 panic value 和调用栈

`panic` 不是普通错误处理手段，I/O 失败、用户输入错误等应用 `error` 返回值处理。`ch5/ch5-09.md`

`Must` 命名约定：包装函数在输入不合法时 panic，用于包级变量初始化：

```go
var httpSchemeRE = regexp.MustCompile(`^https?:`)
```

### 16. recover

只能在 deferred 函数中调用 recover，才能从 panic 恢复。未发生 panic 时 recover 返回 nil：

```go
func Parse(input string) (s *Syntax, err error) {
    defer func() {
        if p := recover(); p != nil {
            err = fmt.Errorf("internal error: %v", p)
        }
    }()
    // ...
}
```

原则：只有选择性地恢复自己明确定义的 panic 类型；不要恢复其他包或回调函数的 panic；公有 API 应返回 error，不应依赖 panic/recover 作为正常控制流。`ch5/ch5-10.md`

## 重点关注点

- bare return 降低可读性，多 return 路径时慎用。
- 错误是预期结果，panic 是程序 bug 的信号。
- 闭包捕获的是变量引用，循环变量陷阱是高频考点。
- defer 参数立即求值，函数体延迟执行；多个 defer 逆序执行。
- 循环中的 defer 不会每次迭代执行，需封装到函数中。
- recover 只在 deferred 函数中有效，且应有选择地使用。
- 函数是一等值，nil 函数值调用会 panic。
