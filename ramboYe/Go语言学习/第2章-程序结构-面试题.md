# 第2章-程序结构-面试题

关联：[[第2章-程序结构]]、[[00-Go语言学习索引]]

## 使用方式
先自己回答，再看参考答案。面试回答不要只背定义，最好补一句“这个规则在代码里会导致什么结果”。

## 基础概念

### 1. Go 中标识符的命名规则是什么？
参考答案：必须以 Unicode 字母或下划线开头，后面可以跟任意数量的字母、数字或下划线。大小写敏感。

### 2. Go 中名字是否导出由什么决定？
参考答案：包级名字如果以大写字母开头就是导出的，可以被其他包访问；小写开头只在当前包内可见。

### 3. Go 的四类主要声明是什么？
参考答案：`var` 声明变量，`const` 声明常量，`type` 声明类型，`func` 声明函数。

### 4. 包级声明和局部声明有什么区别？
参考答案：包级声明在同一个包的所有文件中可见；局部声明只在所在函数或词法块中可见。

### 5. Go 变量没有显式初始化时会怎样？
参考答案：会被初始化为对应类型的零值。Go 不存在未初始化变量。

### 6. 常见类型的零值分别是什么？
参考答案：数值是 `0`，bool 是 `false`，string 是空字符串，指针/slice/map/chan/函数/接口是 `nil`，数组和结构体是各元素或字段的零值。

### 7. `:=` 和 `=` 的区别是什么？
参考答案：`:=` 是短变量声明，用于函数内部声明并初始化局部变量；`=` 是赋值，用于更新已有变量。

### 8. 短变量声明左边必须满足什么条件？
参考答案：同一词法域内，左边至少有一个新变量；否则会编译失败，提示没有新变量。

### 9. 为什么 `f, err := os.Open(...)` 很常见？
参考答案：Go 函数常返回结果和错误两个值，短变量声明可以同时声明文件对象和错误变量，随后立即检查 `err != nil`。

### 10. `new(T)` 返回什么？
参考答案：创建一个 T 类型匿名变量，初始化为 T 的零值，并返回它的地址，类型是 `*T`。

## 指针和生命周期

### 11. `&x` 和 `*p` 分别是什么意思？
参考答案：`&x` 获取变量 x 的地址；`*p` 访问 p 指针指向的变量，也可以用于修改该变量。

### 12. Go 支持指针运算吗？
参考答案：不支持。Go 有指针，但不能像 C 一样对指针做加减运算。

### 13. Go 函数返回局部变量地址安全吗？
参考答案：安全。编译器会通过逃逸分析决定变量存储位置，如果函数返回后还可达，会分配到堆上或保证其生命周期足够长。

### 14. 变量放在栈上还是堆上由 `var` 或 `new` 决定吗？
参考答案：不是。由编译器根据变量是否逃逸等因素决定。

### 15. 作用域和生命周期有什么区别？
参考答案：作用域是编译期概念，描述名字在源代码哪里可见；生命周期是运行期概念，描述变量在内存中存在多久。

### 16. 指针为什么会让程序更难分析？
参考答案：指针会为同一个变量创建别名，变量可能被多个路径读写，状态来源更难追踪。

## 赋值和类型

### 17. 元组赋值的求值顺序是什么？
参考答案：先计算右边所有表达式，再统一赋值给左边变量。所以 `x, y = y, x` 可以正确交换变量。

### 18. Go 中 `++` 和 `--` 是表达式吗？
参考答案：不是，它们是语句。不能写 `x = i++`。

### 19. 多返回值函数通常如何接收？
参考答案：用元组赋值或短变量声明，例如 `f, err := os.Open(name)`。

### 20. `v, ok := m[key]` 中 `ok` 表示什么？
参考答案：表示 map 中是否存在该 key。不存在时 v 是 value 类型零值，ok 是 false。

### 21. 什么是可赋值性？
参考答案：一个值能否赋给某个变量。一般要求类型匹配；`nil` 可以赋给指针和引用类型；常量有更灵活规则。

### 22. `type Celsius float64` 表示什么？
参考答案：声明一个新的命名类型 `Celsius`，底层类型是 `float64`。

### 23. `Celsius` 和 `Fahrenheit` 底层都是 `float64`，能直接相加吗？
参考答案：不能。它们是不同的命名类型，需要显式转换或通过业务函数换算。

### 24. `Celsius(f)` 和 `FToC(f)` 有什么区别？
参考答案：`Celsius(f)` 是类型转换，只改变类型视角，不改变数值；`FToC(f)` 是温度换算，会改变数值。

### 25. 命名类型有什么价值？
参考答案：能表达业务含义、阻止不同概念的数据误混用，还可以为该类型定义方法。

### 26. `String()` 方法和 `fmt` 打印有什么关系？
参考答案：类型实现 `String() string` 后，`fmt` 使用 `%v`、`%s` 或 `Println` 打印时会优先使用这个方法的结果。

## 包、文件和初始化

### 27. 一个 Go 包可以由多个文件组成吗？
参考答案：可以。同一个目录下同一个包名的多个 `.go` 文件共同组成一个包。

### 28. 同一个包中，一个文件的 import 能给另一个文件用吗？
参考答案：不能。import 是文件级作用域，每个文件必须导入自己使用的包。

### 29. 包名和导入路径有什么区别？
参考答案：导入路径是构建工具定位包的字符串，通常全局唯一；包名是在代码中引用该包导出成员时使用的短名字，通常是路径最后一段。

### 30. 包初始化顺序是什么？
参考答案：先初始化依赖包；每个包只初始化一次；包级变量按依赖和声明顺序初始化；再执行 `init`；最后初始化 `main` 包并执行 `main`。

### 31. `init` 函数有什么特点？
参考答案：没有参数和返回值，不能手动调用或引用，程序启动时由运行时自动调用。一个文件可以有多个 `init`。

### 32. `init` 适合做什么？
参考答案：适合做复杂初始化、预计算表格、注册逻辑等。但不应滥用，避免隐藏过多启动副作用。

## 作用域和遮蔽

### 33. Go 编译器查找一个名字时按什么顺序？
参考答案：从最内层词法块向外层查找，直到包级和全局作用域。内层同名声明会遮蔽外层声明。

### 34. 为什么下面代码里的包级 `cwd` 没有被正确赋值？
```go
var cwd string

func init() {
    cwd, err := os.Getwd()
    _ = err
    _ = cwd
}
```
参考答案：`:=` 在 `init` 内声明了新的局部 `cwd` 和 `err`，局部 `cwd` 遮蔽了包级 `cwd`，因此包级变量没有被更新。

### 35. 上题应该如何改？
参考答案：提前声明 `err`，然后使用 `=` 赋值：
```go
var cwd string

func init() {
    var err error
    cwd, err = os.Getwd()
    if err != nil {
        log.Fatal(err)
    }
}
```

### 36. 为什么 Go 推荐错误处理后直接返回，而不是把正常逻辑写在 `else` 里？
参考答案：这样可以减少正常路径缩进，让主逻辑更清晰，也是 Go 常见风格。

### 37. `if f, err := os.Open(name); err != nil { ... }` 中 f 的作用域在哪里？
参考答案：只在该 if 语句及其 else 分支内可见，if 之后不能访问 f。

## 代码阅读题

### 38. 下面代码能编译吗？为什么？
```go
f, err := os.Open("a.txt")
f, err := os.Create("b.txt")
```
参考答案：不能。第二行同一词法域内没有新变量，应该用 `f, err = os.Create("b.txt")`。

### 39. 下面代码输出什么？
```go
x, y := 1, 2
x, y = y, x
fmt.Println(x, y)
```
参考答案：输出 `2 1`。元组赋值先计算右边再统一赋值。

### 40. 下面代码中 `p == q` 通常是什么？
```go
p := new(int)
q := new(int)
fmt.Println(p == q)
```
参考答案：通常是 `false`。每次 `new(int)` 创建新的变量并返回其地址。

### 41. 下面代码为什么不能直接比较？
```go
type Celsius float64
type Fahrenheit float64
var c Celsius
var f Fahrenheit
fmt.Println(c == f)
```
参考答案：`Celsius` 和 `Fahrenheit` 是不同的命名类型，即使底层类型相同，也不能直接比较。

### 42. 下面代码中 `err` 的作用域到哪里？
```go
if err := do(); err != nil {
    return err
}
fmt.Println(err)
```
参考答案：`err` 只在 if 语句内部可见，if 后面的 `fmt.Println(err)` 不能编译。

## 实战追问

### 43. 设计一个温度转换包时，为什么要定义 `Celsius` 和 `Fahrenheit` 两个类型，而不是都用 `float64`？
参考答案：命名类型能表达单位语义，防止把摄氏度和华氏度直接混算，也能为不同类型定义清晰的方法和转换函数。

### 44. 什么情况下你会避免使用 `:=`？
参考答案：当需要给外层变量赋值时，尤其是在 `init`、循环或错误处理逻辑中，避免因为 `:=` 重新声明局部变量造成遮蔽。

### 45. 你如何判断一个包的 API 是否对外暴露过多？
参考答案：检查包级大写名字是否都是外部真正需要的能力；不需要外部使用的类型、变量、函数应小写，隐藏实现细节。

### 46. 为什么每个文件都要求自己导入依赖包？
参考答案：这样文件依赖清晰，编译器能精确检测未使用 import，避免隐式依赖同包其他文件。

### 47. `init` 滥用会有什么问题？
参考答案：启动副作用隐藏，初始化顺序难追踪，测试和复用成本增加。能显式初始化时优先显式初始化。

### 48. 第二章最容易在实际开发中踩的坑是什么？
参考答案：短变量声明导致变量遮蔽，特别是想更新外层变量时误用了 `:=`。其次是混淆类型转换和业务换算。
