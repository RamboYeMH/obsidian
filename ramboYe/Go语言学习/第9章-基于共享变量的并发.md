## 知识点

### 1. 竞争条件

多个 goroutine 并发访问同一变量，且至少一个是写操作，如果没有同步，就可能产生数据竞争。数据竞争会让程序行为不可靠。

### 2. sync.Mutex

`Mutex` 用于保护临界区：

```go
mu.Lock()
defer mu.Unlock()
// read or write shared state
```

锁保护的是不变量，不只是某一行代码。

### 3. sync.RWMutex

读多写少时可以使用 `RWMutex`。多个读锁可并发，写锁独占。不要为了“看起来更高级”滥用，读临界区很短时普通 Mutex 可能更简单。

### 4. 内存同步

编译器和 CPU 可能重排内存读写。锁、channel、`sync.Once` 等同步原语不仅保证互斥，也保证可见性。

### 5. sync.Once

`Once` 保证函数只执行一次，常用于并发安全的惰性初始化。

```go
var loadIconsOnce sync.Once
loadIconsOnce.Do(loadIcons)
```

### 6. race detector

用 `go test -race` 或 `go run -race` 检测运行路径上的数据竞争。它不能证明没有竞争，只能发现测试覆盖到的竞争。

### 7. goroutine 和线程

goroutine 初始栈小、可增长，由 Go runtime 调度到 OS 线程上。`GOMAXPROCS` 控制同时执行 Go 代码的 OS 线程数量。

## 重点关注点

- 共享变量必须明确同步策略。
- 不要复制包含 Mutex 的值。
- 加锁和解锁要覆盖完整不变量。
- `RWMutex` 不是默认选择。
- `go test -race` 是并发代码的基础检查。
