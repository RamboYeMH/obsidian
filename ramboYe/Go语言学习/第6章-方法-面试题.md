# 第6章-方法 面试题

关联：[[第6章-方法]]、[[00-Go语言学习索引]]

---

## Q1：Go 方法和函数的区别是什么？

**答：** 方法是带有接收器参数的函数，接收器将该函数绑定到特定类型。调用时用选择器语法 `p.Distance(q)`，而函数调用是 `Distance(p, q)`。不同类型可以拥有同名方法，各自在各自的命名空间中，不会冲突。

```go
func Distance(p, q Point) float64 { ... }        // 包级函数
func (p Point) Distance(q Point) float64 { ... } // Point 的方法
```

---

## Q2：值接收器和指针接收器如何选择？

**答：** 两个判断条件：

1. **需要修改接收器对象** → 必须用指针接收器
2. **对象较大，想避免每次调用都拷贝** → 用指针接收器

如果类型有任何一个指针接收器方法，约定上所有方法都应统一用指针接收器，避免混用造成困惑。

只读、对象小、不可变的类型（如 `time.Duration`）适合值接收器，调用时安全拷贝。

---

## Q3：编译器在方法调用时的自动转换规则是什么？

**答：** 编译器会在以下情况自动插入取址或解引用：

- 变量 `p`（类型 `T`）调用指针接收器方法：编译器自动插入 `&`，即 `p.ScaleBy(2)` → `(&p).ScaleBy(2)`
- 指针 `pptr`（类型 `*T`）调用值接收器方法：编译器自动插入 `*`，即 `pptr.Distance(q)` → `(*pptr).Distance(q)`

**限制**：只有可寻址的变量才能自动取址。字面量、函数返回值等临时值无法取址：

```go
Point{1, 2}.ScaleBy(2) // 编译错误：无法对字面量取址
```

## Q4：什么是方法提升？嵌入和继承有什么区别？

**答：** 将一个命名类型作为匿名字段嵌入 struct，外层类型自动"提升"内层类型的所有方法，可以直接调用：

```go
type ColoredPoint struct {
    Point        // 嵌入
    Color color.RGBA
}
cp.Distance(q.Point) // 提升来的方法，实际调用 cp.Point.Distance
```

**关键区别**：嵌入是 has-a（组合），不是 is-a（继承）。`ColoredPoint` 不是 `Point`，不能把 `ColoredPoint` 传给接收 `Point` 参数的函数：

```go
p.Distance(q) // 编译错误：ColoredPoint 不是 Point，必须用 q.Point
```

编译器为提升方法生成包装函数，接收器是 `p.Point`，而不是 `p`。

## Q5：nil 可以作为方法的接收器吗？

**答：** 可以，只要方法内部对 nil 的情况做了处理。指针接收器方法尤其常见：

```go
func (list *IntList) Sum() int {
    if list == nil {
        return 0  // nil 代表空链表
    }
    return list.Value + list.Tail.Sum()
}
```

map 类型的方法也可以用 nil 接收器**读取**（返回零值），但**写入** nil map 会 panic。定义允许 nil 接收器的方法时，需要在类型注释中说明 nil 的语义。

## Q6：什么是方法值？什么是方法表达式？有什么区别？

**答：**

**方法值**：`p.Distance` — 选择器不调用方法，返回一个已绑定接收器 `p` 的函数值，调用时只需传方法参数：

```go
f := p.Distance
f(q)  // 等价于 p.Distance(q)
time.AfterFunc(10*time.Second, r.Launch) // 直接传方法值
```

**方法表达式**：`Point.Distance` — 返回一个普通函数，接收器变为第一个参数：

```go
distance := Point.Distance          // func(Point, Point) float64
distance(p, q)                      // 等价于 p.Distance(q)

scale := (*Point).ScaleBy           // func(*Point, float64)
scale(&p, 2)                        // 等价于 p.ScaleBy(2)
```

方法表达式适合在运行时根据条件选择调用哪个方法（用变量存储方法表达式）。

## Q7：为类型定义 String() 方法有什么效果？需要注意什么？

**答：** `fmt` 包在打印值时会检查该值是否实现了 `String() string` 方法，如果实现了则自动调用（依赖接口机制）。这让自定义类型可以控制打印格式。

注意接收器类型：如果 `String()` 定义在 `*T`（指针接收器），则只有打印 `*T` 类型的值时才会自动调用，打印 `T` 类型值时不会：

```go
// *IntSet 有 String() 方法
fmt.Println(&x)         // 自动调用："{1 9 144}"
fmt.Println(x.String()) // 编译器插入 &，自动调用
fmt.Println(x)          // IntSet 无 String()，打印原始结构："{[...]}"
```

如果希望打印 `T` 类型也能自动格式化，应将 `String()` 定义在值接收器上。

## Q8：Go 封装的机制是什么？封装的最小单元是什么？

**答：** Go 唯一的封装手段是**标识符首字母大小写**：大写开头导出（包外可见），小写开头包内可见。

最小封装单元是 **package**，不是类型。同一个包内的所有代码（无论是函数还是方法）都可以访问小写字段，不存在类级别的私有。这意味着要封装一个对象，必须将其定义为 struct，把需要隐藏的字段设为小写。

## Q9：Go 中 getter 方法的命名约定是什么？

**答：** Go 约定 getter 方法省略 `Get` 前缀，直接以字段语义命名；setter 用 `SetXxx`：

```go
// 推荐
func (l *Logger) Flags() int          // 不是 GetFlags
func (l *Logger) SetFlags(flag int)
func (l *Logger) Prefix() string      // 不是 GetPrefix
func (l *Logger) SetPrefix(prefix string)
```

这与 Java 等语言的 `getXxx/setXxx` 约定不同。

## Q10：嵌入 *Point 和嵌入 Point 有什么区别？

**答：**

- **嵌入 `Point`**：外层 struct 中包含一个 Point 的副本，每个对象拥有独立的 Point 数据。
- **嵌入 `*Point`**：外层 struct 持有指向 Point 的指针，多个对象可以共享同一个 Point：

```go
type ColoredPoint struct { *Point; Color color.RGBA }
q.Point = p.Point  // p 和 q 共享同一个 Point
p.ScaleBy(2)       // q.X、q.Y 也随之改变
```

嵌入指针还允许动态改变对象间的关联关系，但需要注意 nil 指针问题（嵌入的 `*Point` 为 nil 时调用方法会 panic）。
