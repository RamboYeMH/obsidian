## 知识点

### 1. 包是工程基本单元

一个目录下的 Go 文件通常属于同一个包。包提供命名空间、封装边界和编译单元。

### 2. 导入路径和包名

导入路径用于定位包，包名用于代码中引用标识符。两者通常相关，但不是一回事。

```go
import "encoding/json"
json.Marshal(v)
```

### 3. import 规则

每个文件必须声明自己用到的 import，不能依赖同包其他文件的导入。未使用的 import 会编译失败。

### 4. 匿名导入

`import _ "pkg"` 只为了执行目标包的 init 副作用，常用于注册数据库驱动、图片格式解码器等。

### 5. internal 包

`internal` 目录限制导入范围，只有父目录树内的代码能导入它，用于表达项目内部 API。

### 6. 常用工具

- `go build`：编译包或命令。
- `go test`：运行测试。
- `go test -run`：按名称筛选测试。
- `go test -bench`：运行基准测试。
- `go doc`：查看文档。
- `go list`：查询包元数据。
- `go fmt`：格式化代码。

现代项目还应熟悉 `go mod tidy`、`go env`、`go version`。

## 重点关注点

- 包名要短小清晰，不要把类型名重复进包名。
- 导出 API 越少越好。
- `internal` 是工程边界工具。
- 匿名导入要谨慎，最好配注释说明用途。
- 工具链是 Go 开发体验的一部分，不是附属知识。
