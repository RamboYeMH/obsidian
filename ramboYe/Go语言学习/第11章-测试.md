## 知识点

### 1. 测试文件和测试函数

测试文件以 `_test.go` 结尾。测试函数格式：

```go
func TestName(t *testing.T) {
}
```

测试失败用 `t.Errorf` 或 `t.Fatalf`。

### 2. 表驱动测试

Go 常用表驱动测试把输入和期望集中到表里：

```go
tests := []struct {
    name string
    in   string
    want string
}{}
for _, tt := range tests {
    t.Run(tt.name, func(t *testing.T) {})
}
```

### 3. 白盒和黑盒测试

同包测试可以访问未导出标识符，外部测试包（如 `mypkg_test`）只能访问导出 API，更接近调用方视角。

### 4. 覆盖率

`go test -cover` 查看覆盖率，`-coverprofile` 生成覆盖率文件。覆盖率高不代表测试质量高，但能发现明显没跑到的路径。

### 5. 基准测试

基准测试函数格式：

```go
func BenchmarkName(b *testing.B) {
    for i := 0; i < b.N; i++ {
    }
}
```

运行：`go test -bench=.`

### 6. 示例函数

`ExampleXxx` 既能进入文档，也能通过 `// Output:` 校验输出。

## 重点关注点

- 测试应验证行为，不要过度绑定内部实现。
- 表驱动测试适合覆盖多组输入。
- `t.Fatal` 会停止当前测试，`t.Error` 继续执行。
- benchmark 要避免把初始化成本算入循环。
- 覆盖率是辅助指标，不是质量指标。
