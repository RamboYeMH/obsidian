## 知识点

### 1. 类型分类
Go 的数据类型可以分为基础类型、复合类型、引用类型和接口类型。本章只讲基础类型：数字、字符串、布尔型。复合类型从第 4 章开始。

### 2. 整型
Go 提供固定宽度整数：`int8/int16/int32/int64`、`uint8/uint16/uint32/uint64`，也提供平台相关宽度的 `int` 和 `uint`。`rune` 等价于 `int32`，通常表示 Unicode 码点；`byte` 等价于 `uint8`，通常表示原始字节。

`int` 和 `int32` 是不同类型，即使在某些平台上宽度相同，也需要显式转换。

### 3. 整数溢出
整数运算溢出时，高位会被丢弃。例如 `uint8(255)+1` 得到 `0`，`int8(127)+1` 得到 `-128`。这不是 panic，而是按类型宽度截断。来源：`ch3/ch3-01.md:36`。

### 4. 位运算
常用位运算：

```go
&   // AND
|   // OR
^   // XOR；一元时表示按位取反
&^  // AND NOT，按位置零
<<  // 左移
>>  // 右移
```

无符号数更适合作为纯 bit 模式处理。有符号数右移会用符号位填充，高位行为不适合做通用 bit 集合。

### 5. 尽量用 `int` 表示普通数量
即使数量不可能为负，也通常使用 `int`，因为 `len` 返回 `int`。滥用 `uint` 容易写出逆序循环 bug：`i >= 0` 对无符号整数永远为真。

### 6. 显式类型转换
Go 不会在不同整数类型之间自动转换：

```go
var apples int32 = 1
var oranges int16 = 2
var compote = int(apples) + int(oranges)
```

浮点转整数会丢弃小数部分并向 0 截断。不要对超出目标类型范围的值做转换，结果可能依赖实现。

### 7. 浮点数
Go 有 `float32` 和 `float64`。一般优先用 `float64`，因为 `float32` 精度低，累计误差容易扩散。`float32` 大约 6 位十进制精度，`float64` 大约 15 位。

浮点特殊值：`+Inf`、`-Inf`、`NaN`。`NaN` 和任何值都不相等，包括它自己，所以不要用 `x == math.NaN()` 判断，应使用 `math.IsNaN(x)`。

如果函数可能计算失败，通常返回 `(value, ok)` 或 `(value, error)`，不要只靠 NaN 表达失败。

### 8. 复数
Go 支持 `complex64` 和 `complex128`。使用 `complex(real, imag)` 构造，`real` 和 `imag` 取实部和虚部。

```go
x := 1 + 2i
y := 3 + 4i
fmt.Println(x * y)
```

`math/cmplx` 提供复数数学函数。Mandelbrot 示例在 `vendor/gopl.io/ch3/mandelbrot/main.go:18`。

### 9. 布尔值
布尔值只有 `true` 和 `false`。`&&` 和 `||` 有短路行为，因此下面写法安全：

```go
s != "" && s[0] == 'x'
```

Go 不允许 bool 和数字隐式互转，不能把 `0/1` 当作 `false/true`。

### 10. 字符串是不可变字节序列
字符串本质是只读字节序列，不一定是字符序列。`len(s)` 返回字节数，不是 rune 数；`s[i]` 返回第 i 个字节。字符串不能原地修改：`s[0] = 'L'` 不能编译。

字符串切片 `s[i:j]` 也是按字节切片，不是按字符切片。它通常和原字符串共享底层数据，代价低。

### 11. UTF-8、Unicode 和 rune
Go 源文件采用 UTF-8。`rune` 是 Unicode 码点，等价于 `int32`。UTF-8 是变长编码，一个 rune 可能占 1 到 4 个字节。

```go
s := "Hello, 世界"
len(s)                    // 13 字节
utf8.RuneCountInString(s) // 9 个 rune
```

遍历字符串时，`range` 会自动按 UTF-8 解码，索引是字节偏移，值是 rune。

### 12. `[]rune` 和 `[]byte`
需要按字符处理时，可以转为 `[]rune`；需要处理原始字节或构建二进制数据时，用 `[]byte`。

```go
r := []rune(s)
b := []byte(s)
```

`[]byte(s)` 和 `string(b)` 通常会复制数据，以保证字符串不可变。

### 13. 字符串相关包
- `strings`：字符串查询、替换、分割、连接。
- `bytes`：针对 `[]byte` 的类似操作。
- `strconv`：字符串和数字、布尔值互转。
- `unicode`：按 Unicode 规则判断 rune 分类、大小写转换。
- `unicode/utf8`：UTF-8 编解码。

构建大量字符串时优先考虑 `bytes.Buffer` 或后续会学到的 builder 类工具，避免反复 `+` 造成分配。

### 14. 常量
常量在编译期计算。常量可以是布尔、字符串或数字。常量不可修改，适合表达数学常量、枚举值、数组长度等。

常量运算也在编译期完成，因此某些错误能提前发现，例如常量除零、常量字符串索引越界、常量溢出。

### 15. `iota`
`iota` 在一个 `const` 声明块中从 0 开始，每一行常量声明加 1。常用于枚举和 bit flag。

```go
type Weekday int
const (
    Sunday Weekday = iota
    Monday
    Tuesday
)
```

bit flag 示例：`vendor/gopl.io/ch3/netflag/netflag.go:15`。

### 16. 无类型常量
Go 的许多常量是无类型常量，编译器会用更高精度计算，直到它被赋给变量或用于需要具体类型的上下文。

默认类型规则：

```go
i := 0      // int
r := '\000' // rune，即 int32
f := 0.0    // float64
c := 0i     // complex128
```

这解释了为什么 `math.Pi` 可以直接赋给 `float32`、`float64` 或 `complex128`。

## 重点关注点
- `len(string)` 是字节数，不是字符数。
- `range string` 的索引是字节偏移，值是 rune。
- 字符串不可变，`[]byte` 可变；两者转换通常复制。
- `int` 和 `int32` 是不同类型，算术运算要求类型一致。
- 普通数量优先使用 `int`，不要为了“非负”滥用 `uint`。
- 整数溢出不会 panic；浮点 NaN 不能用 `==` 判断。
- `bool` 不会隐式转换为 `0/1`。
- `iota` 适合枚举和 bit flag，不适合所有序列生成。
- 类型转换和数值格式化/解析不是一回事，字符串和数字互转用 `strconv`。
