# 第10章-包和工具 面试题

关联：[[第10章-包和工具]]、[[00-Go语言学习索引]]

## Q1：Go 中 package 的作用是什么？

**答：** package 是命名空间、封装边界、复用单位和编译单位。

## Q2：导入路径和包名有什么区别？

**答：** 导入路径用于定位包，包名用于代码中引用。二者通常一致但不是同一个概念。

## Q3：一个文件能使用同包其他文件 import 的包吗？

**答：** 不能。每个 Go 文件必须声明自己直接使用的 import。

## Q4：未使用的 import 为什么会报错？

**答：** Go 强制代码保持干净，避免无效依赖和编译成本。

## Q5：匿名导入 `_ "pkg"` 的用途是什么？

**答：** 只执行目标包的 init 副作用，不直接引用包名。常见于驱动或格式注册。

## Q6：internal 包有什么限制？

**答：** 只有 internal 父目录树内的代码可以导入该包，父目录之外不能导入。

## Q7：Go 包命名有什么建议？

**答：** 短小、清晰、小写，不用下划线，避免和导出类型重复。例如 `bytes.Buffer`，而不是 `bytes.BytesBuffer`。

## Q8：`go test ./...` 表示什么？

**答：** 运行当前模块或目录下所有子包的测试。

## Q9：`go list` 有什么用？

**答：** 查询包的导入路径、依赖、源码文件、测试文件等元数据，适合脚本和工具使用。

## Q10：为什么要控制包的导出 API？

**答：** 导出的标识符会成为对外契约，越多越难维护和演进。
