## 知识点

### 1. 数组
数组是固定长度的同类型元素序列。长度是数组类型的一部分，所以 `[3]int` 和 `[4]int` 是不同类型。数组长度必须是常量表达式。

数组默认按元素类型零值初始化，可以用字面量初始化：

```go
q := [...]int{1, 2, 3}
r := [...]int{99: -1}
```

如果数组元素可比较，那么数组也可比较，比较时逐元素比较。`sha256.Sum256` 返回 `[32]byte`，可以直接比较摘要是否相等。示例：`vendor/gopl.io/ch4/sha256/main.go:14`。

函数参数是值传递，数组作为参数会复制整个数组。大数组应避免直接传值，通常用 slice，少数场景可传数组指针。

### 2. Slice 的结构
slice 是变长序列，类型写作 `[]T`。slice 不是数组本身，它是对底层数组某个区间的描述，包含三部分：指针、长度、容量。

```go
len(s) // 当前元素数
cap(s) // 从起始位置到底层数组末尾的容量
```

多个 slice 可以共享同一个底层数组，修改一个 slice 的元素可能影响另一个 slice。

### 3. 切片表达式
`s[i:j]` 左闭右开，创建新 slice。对 slice 来说，`j` 可以超过 `len(s)` 但不能超过 `cap(s)`，否则 panic。对字符串来说，切片按字节操作。

### 4. Slice 和 nil
slice 零值是 nil：长度和容量都为 0，且没有底层数组。

```go
var s []int      // nil slice
s = []int{}      // 非 nil，但长度为 0
```

判断是否为空，应使用 `len(s) == 0`，不要用 `s == nil`。除非文档明确区分，否则函数应同等对待 nil slice 和空 slice。

### 5. Slice 不可比较
slice 只能和 nil 比较，不能用 `==` 比较两个 slice 的内容。原因包括：slice 间接引用底层数组、元素会变、可能包含自身。比较内容需要手写循环或使用专用函数，例如 `bytes.Equal`。

### 6. `append`
`append` 追加元素并返回新的 slice。必须接收返回值：

```go
s = append(s, x)
```

如果容量足够，append 会复用底层数组；容量不足，会分配新数组并复制旧元素。不要依赖 append 后是否共享底层数组。示例：`vendor/gopl.io/ch4/append/main.go:32`。

`append` 可以追加多个元素，也可以用 `...` 追加另一个 slice：

```go
s = append(s, 1, 2, 3)
s = append(s, other...)
```

### 7. Slice 原地算法
slice 适合做原地过滤、反转、栈、删除等操作。例如 `nonempty` 会复用原底层数组，返回过滤后的前缀：`vendor/gopl.io/ch4/nonempty/main.go:15`。

常见模式：

```go
out := s[:0]
for _, x := range s {
    if keep(x) {
        out = append(out, x)
    }
}
```

这种写法高效，但会覆盖原底层数组，调用方通常要写 `s = filter(s)`。

### 8. Map
map 是哈希表引用，类型写作 `map[K]V`。key 类型必须可比较，value 类型不限。slice、map、函数不能作为 key。

```go
ages := make(map[string]int)
ages["alice"] = 31
```

查找不存在的 key 会返回 value 类型零值。需要区分“不存在”和“存在但值为零”时，使用 ok idiom：

```go
v, ok := m[key]
```

### 9. nil map
map 零值是 nil。nil map 可以查找、删除、取长度、range，但不能写入；写入会 panic。写入前必须 `make` 或使用 map 字面量创建。

### 10. Map 遍历无序
map 的 range 顺序不确定，且会变化。需要稳定输出时，先收集 key，再排序，然后按 key 访问 map。

### 11. Map 不能比较
map 只能和 nil 比较。比较两个 map 内容必须手写循环，并用 ok 区分 key 不存在和值为零。

### 12. 用 map 实现 set
Go 没有内置 set，常用 `map[T]bool` 或 `map[T]struct{}` 表示集合。`map[T]bool` 写法简单，`map[T]struct{}` 更强调只关心 key。

### 13. 结构体
结构体是由零个或多个字段组成的聚合类型。字段通过点操作访问。结构体零值是所有字段的零值。

结构体字段大写开头表示导出，小写开头表示包内可见。字段顺序是结构体类型的一部分。

结构体可以包含指向自身类型的指针字段，从而构建链表、树等递归结构。示例：`vendor/gopl.io/ch4/treesort/sort.go:10`。

### 14. 结构体字面值
两种常见写法：

```go
Point{1, 2}
Point{X: 1, Y: 2}
```

按顺序写法依赖字段顺序，通常只在包内或小结构体中使用。带字段名写法更稳定，也可只初始化部分字段，其余为零值。

大结构体通常用指针传递；如果函数要修改结构体字段，必须传指针。

### 15. 结构体比较和 map key
如果结构体全部字段都可比较，则结构体可比较，可以作为 map key。相等比较逐字段比较。

### 16. 结构体嵌入
匿名字段允许把一个命名类型嵌入结构体。外层结构体可以通过简短点操作访问嵌入字段的字段和方法。

```go
type Circle struct { Point; Radius int }
type Wheel struct { Circle; Spokes int }
w.X = 8 // 等价于 w.Circle.Point.X = 8
```

嵌入是 Go 组合思想的重要基础，不是继承。示例：`vendor/gopl.io/ch4/embed/main.go:11`。

### 17. JSON
`encoding/json` 用于 JSON 编解码。`json.Marshal` 将 Go 值编码为 JSON，`json.Unmarshal` 将 JSON 解码到 Go 值。

只有导出的结构体字段会被 JSON 编码/解码。字段 tag 可以控制 JSON 字段名和选项：

```go
Year  int  `json:"released"`
Color bool `json:"color,omitempty"`
```

`omitempty` 表示字段为零值时省略。示例：`vendor/gopl.io/ch4/movie/main.go:16`。

网络响应推荐用 `json.Decoder` 流式解码：`vendor/gopl.io/ch4/github/search.go:17`。

### 18. 模板
`text/template` 和 `html/template` 用于复杂文本/HTML 输出。模板 action 写在 `{{...}}` 中，`.` 表示当前数据。

`html/template` 会根据上下文自动转义，能防止 HTML 注入。输出 HTML 时应优先使用 `html/template`，不要用 `text/template` 直接拼 HTML。

可以用 `template.FuncMap` 注册自定义函数，用 `template.Must` 简化模板初始化错误处理。示例：`vendor/gopl.io/ch4/issuesreport/main.go:37`、`vendor/gopl.io/ch4/issueshtml/main.go:19`。

## 重点关注点
- 数组是值，slice 是描述底层数组的三元结构。
- 数组长度是类型的一部分，`[3]int` 和 `[4]int` 不兼容。
- slice 共享底层数组，修改元素可能影响其他 slice。
- `append` 必须接收返回值，且不要假设是否扩容。
- 判断 slice 为空用 `len(s) == 0`。
- map 的 key 必须可比较，map 遍历顺序不稳定。
- nil map 能读不能写；nil slice 可以 append。
- map/slice 不能直接比较内容。
- 结构体字段导出规则影响 JSON 编码。
- JSON tag 用反引号原生字符串，`omitempty` 是高频考点。
- `html/template` 自动转义，安全性优于手写拼接 HTML。
