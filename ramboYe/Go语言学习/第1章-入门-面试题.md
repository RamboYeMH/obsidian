# 第1章-入门-面试题

关联：[[第1章-入门]]、[[00-Go语言学习索引]]

## 使用方式
先遮住参考答案自己回答。回答时尽量说清楚：概念是什么、为什么这样设计、代码里会怎么写、常见坑是什么。

## 基础概念

### 1. `package main` 有什么特殊含义？
参考答案：`package main` 表示当前包会被编译成可执行程序。可执行程序必须包含 `main` 包，并且通常需要有 `func main()` 作为程序入口。

### 2. `func main()` 的作用是什么？
参考答案：它是 Go 可执行程序启动后执行的入口函数。程序启动时先初始化包级变量和 `init` 函数，然后执行 `main.main()`。

### 3. Go 中未使用的 import 为什么会编译失败？
参考答案：Go 强制保持代码干净，未使用的 import 通常意味着代码遗留或依赖错误。编译失败可以避免无效依赖和隐藏问题。

### 4. `go run` 和 `go build` 有什么区别？
参考答案：`go run` 会临时编译并直接运行程序，适合快速验证；`go build` 会生成可执行文件，适合构建产物和后续部署运行。

### 5. Go 为什么推荐使用 `gofmt`？
参考答案：`gofmt` 统一代码格式，减少风格争论，也让工具能更稳定地做代码分析和自动转换。

### 6. `os.Args` 是什么？`os.Args[0]` 表示什么？
参考答案：`os.Args` 是 `[]string`，保存命令行参数。`os.Args[0]` 是程序自身的命令名，真正传入的参数通常从 `os.Args[1:]` 开始。

### 7. Go 的 slice 表达式 `s[m:n]` 是什么区间？
参考答案：左闭右开，包含 `m`，不包含 `n`。结果长度是 `n-m`。

### 8. `:=` 和 `var` 声明有什么区别？
参考答案：`:=` 是短变量声明，只能在函数内部使用，并由初始值推断类型；`var` 可以在函数内或包级使用，可以显式指定类型，也可以依赖零值初始化。

### 9. 空标识符 `_` 的用途是什么？
参考答案：用于丢弃不需要的值。例如 `for _, arg := range os.Args[1:]` 中丢弃索引，只保留元素值。

### 10. Go 里 `i++` 和 C/Java 里的 `i++` 有什么不同？
参考答案：Go 的 `i++` 是语句，不是表达式，不能写 `j := i++`。Go 也不支持前置自增 `++i`。

## 字符串、输入和文件

### 11. 为什么大量字符串拼接不推荐反复使用 `s += x`？
参考答案：字符串不可变，每次拼接都可能产生新字符串，数据量大时开销较高。可以用 `strings.Join` 或 `strings.Builder`。

### 12. `strings.Join(os.Args[1:], " ")` 做了什么？
参考答案：把参数 slice 中的字符串用空格连接成一个字符串，比手动循环拼接更简洁，也更适合批量拼接。

### 13. `bufio.Scanner` 适合解决什么问题？
参考答案：适合按行或按词扫描输入，例如从标准输入或文件中逐行读取文本。

### 14. `map[string]int` 常用于什么场景？
参考答案：常用于按字符串计数、查找、去重等场景。第一章的 `dup` 程序用它统计每一行出现次数。

### 15. 访问 `map` 中不存在的 key 会发生什么？
参考答案：会返回 value 类型的零值。例如 `map[string]int` 中不存在的 key 返回 `0`，因此可以直接 `counts[line]++`。

### 16. Go 遍历 map 的顺序稳定吗？
参考答案：不稳定，不能依赖 map 的遍历顺序。如果需要稳定输出，要先取出 key 排序。

### 17. `os.Open` 为什么通常要检查 `err != nil`？
参考答案：文件可能不存在、权限不足或路径错误。Go 使用显式错误返回，调用者必须检查错误并决定是否继续。

### 18. 打开文件后为什么要调用 `Close()`？
参考答案：释放文件描述符等系统资源，避免资源泄漏。长时间运行的程序尤其需要注意。

## HTTP 和资源管理

### 19. `http.Get(url)` 返回什么？
参考答案：返回 `*http.Response` 和 `error`。如果请求成功，响应体在 `resp.Body` 中；如果失败，`err` 描述失败原因。

### 20. 为什么读取 HTTP 响应后要关闭 `resp.Body`？
参考答案：关闭响应体可以释放网络连接相关资源，也有助于连接复用。忘记关闭会造成资源泄漏。

### 21. `io.ReadAll(resp.Body)` 和 `io.Copy(os.Stdout, resp.Body)` 有什么区别？
参考答案：`io.ReadAll` 会把响应体全部读到内存；`io.Copy` 可以流式复制到目标 writer，更适合大响应体。

### 22. `fmt.Fprintf(os.Stderr, ...)` 和 `fmt.Printf(...)` 有什么区别？
参考答案：`Fprintf` 可以指定输出目标，`os.Stderr` 表示标准错误；`Printf` 默认输出到标准输出。

## 并发和 Web 服务

### 23. goroutine 是什么？
参考答案：goroutine 是 Go 中轻量级的并发执行单元。使用 `go f()` 可以启动一个新的 goroutine 并发执行函数。

### 24. channel 的作用是什么？
参考答案：channel 用于 goroutine 之间传递数据和同步。发送使用 `ch <- value`，接收使用 `<-ch`。

### 25. 无缓冲 channel 的发送和接收有什么特点？
参考答案：发送方会阻塞直到有接收方接收，接收方也会阻塞直到有发送方发送，因此它同时完成数据传递和同步。

### 26. `fetchall` 为什么比逐个请求 URL 更快？
参考答案：它为每个 URL 启动 goroutine 并发请求，总耗时通常接近最慢请求的耗时，而不是所有请求耗时之和。

### 27. `http.HandleFunc("/", handler)` 做了什么？
参考答案：把 URL pattern `/` 和处理函数 `handler` 绑定起来。请求到达时，HTTP 服务会调用对应 handler 生成响应。

### 28. `http.ResponseWriter` 是什么？
参考答案：它是写 HTTP 响应的接口。可以把响应内容通过 `fmt.Fprintf(w, ...)` 写给客户端。

### 29. `*http.Request` 里通常可以拿到哪些信息？
参考答案：请求方法、URL、协议版本、Header、Host、RemoteAddr、Form 参数、Body 等。

### 30. Web 服务里的共享计数器为什么需要 `sync.Mutex`？
参考答案：HTTP 服务会并发处理请求，多个 goroutine 同时修改共享变量会产生竞态条件。`Mutex` 可以保证同一时间只有一个 goroutine 修改计数器。

## 代码阅读题

### 31. 下面代码输出什么？为什么？
```go
args := []string{"cmd", "a", "b", "c"}
fmt.Println(args[1:3])
```
参考答案：输出 `[a b]`。slice 表达式左闭右开，包含索引 1 和 2，不包含索引 3。

### 32. 下面代码有什么问题？
```go
for i := 0; i < len(os.Args); i++ {
    j := i++
    fmt.Println(j)
}
```
参考答案：`i++` 在 Go 中是语句，不是表达式，不能赋值给 `j`。这段代码不能编译。

### 33. 下面代码为什么可以统计不存在的 key？
```go
counts := make(map[string]int)
counts["go"]++
fmt.Println(counts["go"])
```
参考答案：不存在的 `"go"` key 初始读取为 `int` 零值 `0`，执行 `++` 后变成 `1`。

### 34. 下面代码中 `_` 的作用是什么？
```go
for _, arg := range os.Args[1:] {
    fmt.Println(arg)
}
```
参考答案：`range` 会返回索引和值。这里不需要索引，所以用 `_` 丢弃，只使用参数值。

### 35. 下面 handler 的响应内容写到了哪里？
```go
func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "URL.Path = %q\n", r.URL.Path)
}
```
参考答案：写到了 HTTP 响应中，客户端浏览器或 HTTP 客户端会收到这段内容。

## 实战追问

### 36. 如果让你改造 `fetch`，自动给缺少协议的 URL 添加 `http://`，你会怎么做？
参考答案：使用 `strings.HasPrefix` 判断是否以 `http://` 或 `https://` 开头；如果都不是，则拼接 `http://`。

### 37. 如果让 `dup2` 输出重复行出现在哪些文件中，数据结构该怎么设计？
参考答案：可以使用 `map[string]map[string]int`，外层 key 是行内容，内层 key 是文件名，value 是该文件内出现次数。也可以使用结构体保存总次数和文件集合。

### 38. 如果 `fetchall` 中某个网站一直不响应，程序会怎样？
参考答案：对应 goroutine 会长时间阻塞在请求或读取阶段，主 goroutine 等待 channel 结果时也可能一直等下去。后续可以用超时、context 或自定义 HTTP client 处理。

### 39. 为什么第一章说 `io.Writer` 很重要？
参考答案：因为它把“写到哪里”抽象成统一接口。同一段生成内容的代码可以写到标准输出、文件、HTTP 响应或丢弃目标。

### 40. 你会如何判断第一章是否真正学会？
参考答案：能独立写出命令行程序、文件读取程序、HTTP GET 程序、最小 Web 服务；能解释错误处理、资源关闭、map 计数、goroutine/channel 的基本配合。
