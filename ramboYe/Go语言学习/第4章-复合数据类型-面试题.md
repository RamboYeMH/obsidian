# 第4章-复合数据类型-面试题

关联：[[第4章-复合数据类型]]、[[00-Go语言学习索引]]

## 数组和 Slice

### 1. 数组和 slice 的核心区别是什么？
参考答案：数组长度固定，长度是类型的一部分，数组是值语义；slice 长度可变，是对底层数组的引用描述，包含指针、长度和容量。

### 2. `[3]int` 和 `[4]int` 是同一种类型吗？
参考答案：不是。数组长度是类型的一部分。

### 3. 数组作为函数参数会发生什么？
参考答案：会复制整个数组。函数内修改的是副本，不影响调用者原数组，除非传数组指针。

### 4. 数组可以用 `==` 比较吗？
参考答案：如果元素类型可比较，数组可以比较，比较时逐元素比较。

### 5. slice 由哪三部分组成？
参考答案：指向底层数组元素的指针、长度 `len`、容量 `cap`。

### 6. 多个 slice 之间可能互相影响吗？
参考答案：可能。它们可能共享同一个底层数组，修改元素会被其他共享该元素的 slice 看到。

### 7. `s[i:j]` 的边界规则是什么？
参考答案：左闭右开。对 slice 来说，`j` 不能超过 `cap(s)`；超过 `len(s)` 但不超过 `cap(s)` 可以扩展 slice。

### 8. slice 可以用 `==` 比较吗？
参考答案：不能，除了和 `nil` 比较。比较内容需要手写循环或使用专用函数。

### 9. nil slice 和空 slice 有什么区别？
参考答案：nil slice 没有底层数组，`s == nil`；空 slice 长度为 0 但可能非 nil，例如 `[]int{}`。判断空应使用 `len(s)==0`。

### 10. nil slice 可以 append 吗？
参考答案：可以。`append` 会分配底层数组并返回新 slice。

### 11. 为什么 `append` 的返回值必须接住？
参考答案：append 可能改变 slice 的长度、容量和底层数组地址，必须用返回值更新 slice 描述符。

### 12. append 后原 slice 和新 slice 一定共享底层数组吗？
参考答案：不一定。容量足够时可能共享，容量不足时会分配新数组。

### 13. 如何原地过滤 slice？
参考答案：常用 `out := s[:0]`，遍历原 slice，把保留元素 append 到 out，最后返回 out。

### 14. 如何用 slice 模拟栈？
参考答案：`append` 入栈，`top := stack[len(stack)-1]` 查看栈顶，`stack = stack[:len(stack)-1]` 出栈。

## Map

### 15. map 的 key 类型有什么要求？
参考答案：key 必须可比较，支持 `==`。slice、map、function 不能作为 key。

### 16. map 查找不存在的 key 会怎样？
参考答案：返回 value 类型的零值。

### 17. 如何区分 key 不存在和值为零？
参考答案：使用 ok idiom：`v, ok := m[key]`。

### 18. nil map 可以做哪些操作？
参考答案：可以查找、删除、取 len、range；不能写入，写入会 panic。

### 19. map 遍历顺序稳定吗？
参考答案：不稳定，Go 故意让 map 遍历顺序不确定，不能依赖。

### 20. 如何稳定输出 map？
参考答案：先收集 key 到 slice，排序 key，再按排序后的 key 访问 map。

### 21. map 可以比较吗？
参考答案：不能，除了和 nil 比较。比较内容要手写循环，并用 ok 区分缺失 key。

### 22. Go 如何实现 set？
参考答案：常用 `map[T]bool` 或 `map[T]struct{}`。key 表示集合元素。

### 23. 为什么不能对 map 元素取地址？
参考答案：map 扩容时元素可能迁移，地址会失效，所以 Go 禁止 `&m[key]`。

## 结构体

### 24. 结构体是什么？
参考答案：结构体是由零个或多个字段组成的聚合类型，字段可以是不同类型。

### 25. 结构体零值是什么？
参考答案：每个字段都是对应类型的零值。

### 26. 结构体字段顺序重要吗？
参考答案：重要。字段顺序是结构体类型的一部分，也影响按顺序写的结构体字面值。

### 27. 结构体字段如何导出？
参考答案：字段名大写开头导出，小写开头只在包内可见。

### 28. 结构体字面值有哪两种常见写法？
参考答案：按字段顺序写 `Point{1, 2}`，或按字段名写 `Point{X:1, Y:2}`。后者更稳定。

### 29. 什么时候结构体需要传指针？
参考答案：结构体较大避免复制时，或者函数需要修改结构体字段时。

### 30. 结构体能比较吗？
参考答案：如果所有字段都可比较，则结构体可比较，并可作为 map key。

### 31. 空结构体 `struct{}` 有什么用途？
参考答案：大小为 0，常用于表示不携带额外信息的值，例如 `map[T]struct{}` 实现集合。

### 32. 结构体嵌入是什么？
参考答案：匿名字段嵌入一个命名类型，外层可以直接访问嵌入字段的字段和方法。它是组合，不是继承。

### 33. 嵌入字段的字面值可以直接写提升字段吗？
参考答案：不能。结构体字面值必须按实际结构写，例如 `Wheel{Circle: Circle{Point: Point{X:8,Y:8}}}`。

## JSON 和模板

### 34. `json.Marshal` 和 `json.Unmarshal` 分别做什么？
参考答案：`Marshal` 把 Go 值编码为 JSON 字节；`Unmarshal` 把 JSON 字节解码到 Go 值。

### 35. JSON 编码结构体时，哪些字段会被处理？
参考答案：只有导出的结构体字段会被编码和解码。

### 36. JSON tag 有什么作用？
参考答案：控制 JSON 字段名和编码选项，例如 `json:"released"` 改名，`omitempty` 零值时省略。

### 37. `omitempty` 会省略哪些值？
参考答案：字段为零值时省略，例如 false、0、空字符串、nil 指针、nil slice/map 等。

### 38. 为什么 HTTP JSON 响应常用 `json.Decoder`？
参考答案：它能直接从 `io.Reader` 流式解码，适合网络响应体，不需要先完整读入内存。

### 39. 解码 JSON 时 Go 结构体可以只声明部分字段吗？
参考答案：可以。未声明或不关心的 JSON 字段会被忽略。

### 40. `text/template` 和 `html/template` 的区别是什么？
参考答案：API 类似，但 `html/template` 会根据 HTML/JS/CSS/URL 上下文自动转义，更适合生成 HTML，能降低注入风险。

### 41. 模板中的 `.` 表示什么？
参考答案：当前数据对象。执行模板时初始 `.` 是传给 `Execute` 的数据，`range` 中会变成当前元素。

### 42. `template.Must` 做什么？
参考答案：如果传入 error 非 nil 就 panic，否则返回模板对象。常用于程序启动阶段解析固定模板。

## 代码阅读题

### 43. 下面输出什么？
```go
a := [...]int{1, 2, 3}
fmt.Printf("%T", a)
```
参考答案：`[3]int`。

### 44. 下面代码能编译吗？
```go
q := [3]int{1, 2, 3}
q = [4]int{1, 2, 3, 4}
```
参考答案：不能。`[3]int` 和 `[4]int` 是不同类型。

### 45. 下面代码为什么要写回 s？
```go
s = append(s, x)
```
参考答案：append 可能返回新的 slice 描述符，不写回会丢失新长度或新底层数组。

### 46. 下面代码有什么问题？
```go
var m map[string]int
m["x"] = 1
```
参考答案：nil map 不能写入，会 panic。应先 `make(map[string]int)`。

### 47. 下面代码为什么不能编译？
```go
ages := map[string]int{"bob": 1}
_ = &ages["bob"]
```
参考答案：map 元素不可取地址，因为 map 可能扩容迁移元素。

### 48. 下面比较结果是什么？
```go
type Point struct{ X, Y int }
fmt.Println(Point{1,2} == Point{1,2})
```
参考答案：`true`，因为所有字段都是可比较的 int。

### 49. 下面 JSON 会输出 Color 字段吗？
```go
type Movie struct { Color bool `json:"color,omitempty"` }
json.Marshal(Movie{})
```
参考答案：不会。Color 是 false，为零值，`omitempty` 会省略。

### 50. 下面 map 比较为什么不能只判断 `xv != y[k]`？
参考答案：因为 y 中缺失 key 时 `y[k]` 返回零值，可能和 xv 相同，导致把不同 map 误判为相等。

## 实战追问

### 51. 什么时候用数组，什么时候用 slice？
参考答案：固定大小且大小是语义一部分时用数组，例如哈希摘要 `[32]byte`；大多数动态序列用 slice。

### 52. 如何避免 slice 共享底层数组带来的意外修改？
参考答案：需要独立数据时显式复制，例如 `dst := append([]T(nil), src...)` 或 `copy` 到新 slice。

### 53. 如何设计一个稳定的 JSON API 结构体？
参考答案：导出需要编解码的字段，使用 JSON tag 固定字段名，谨慎使用 `omitempty`，避免把内部字段暴露出去。

### 54. 为什么说结构体嵌入不是继承？
参考答案：嵌入是组合和字段/方法提升，外层不是内层的子类型；Go 没有类继承体系。

### 55. 生成 HTML 为什么不应该手动拼字符串？
参考答案：容易遗漏转义导致 HTML 注入。应使用 `html/template` 自动按上下文转义。

### 56. 第 4 章最容易踩的坑是什么？
参考答案：slice 共享底层数组、append 后忘记接返回值、nil map 写入 panic、map 遍历顺序不稳定、JSON 字段未导出导致不编码。
