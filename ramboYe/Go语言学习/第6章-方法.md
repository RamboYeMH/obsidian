## 知识点

### 1. 方法声明

方法是在函数名前加一个接收器参数。接收器参数将该函数绑定到特定类型：

```go
func (p Point) Distance(q Point) float64 {
    return math.Hypot(q.X-p.X, q.Y-p.Y)
}
```

接收器名通常用类型名首字母（如 `p` for `Point`），不用 `this` 或 `self`。调用时用选择器语法：`p.Distance(q)`。`ch6/ch6-01.md:1`

不同类型可以有相同方法名，各自在自己的命名空间里。可以给同一包内任意命名类型定义方法，但底层类型不能是指针或 interface。

### 2. 指针接收器

需要修改接收器对象、或对象较大想避免拷贝时，使用指针接收器：

```go
func (p *Point) ScaleBy(factor float64) {
    p.X *= factor
    p.Y *= factor
}
```

**编译器自动转换规则**（三种合法调用形式）：

| 接收器实参 | 接收器形参 | 编译器行为 |
|---|---|---|
| `T` | `T` | 直接匹配 |
| `*T` | `*T` | 直接匹配 |
| `T`（变量） | `*T` | 隐式取址 `(&p).ScaleBy(2)` |
| `*T` | `T` | 隐式解引用 `(*pptr).Distance(q)` |

注意：对临时值（字面量）无法取址，`Point{1,2}.ScaleBy(2)` 编译报错。

**约定**：如果类型有任意一个指针接收器方法，则所有方法都应使用指针接收器。`ch6/ch6-02.md`

### 3. nil 是合法接收器

指针接收器的方法可以接受 nil，需要在类型注释中说明 nil 的语义：

```go
// A nil *IntList represents the empty list.
func (list *IntList) Sum() int {
    if list == nil {
        return 0
    }
    return list.Value + list.Tail.Sum()
}
```

map 类型方法也可以用 nil 接收器读取（返回零值），但写入 nil map 会 panic。`ch6/ch6-02.md`

### 4. 结构体嵌入与方法提升

匿名字段嵌入后，外层类型自动获得内层类型的所有字段和方法，无需显式委托：

```go
type ColoredPoint struct {
    Point        // 嵌入，不是字段名
    Color color.RGBA
}

cp.X = 1              // 等价于 cp.Point.X = 1
cp.Distance(q.Point)  // 调用提升来的 Point.Distance
cp.ScaleBy(2)         // 调用提升来的 (*Point).ScaleBy
```

**重要**：这是 has-a（组合），不是 is-a（继承）。`ColoredPoint` 不是 `Point`，不能把 `ColoredPoint` 传给接受 `Point` 的函数。编译器为提升的方法生成包装函数，实际上把 `p.Point` 作为接收器传入。`ch6/ch6-03.md`

也可以嵌入指针类型 `*Point`，允许多个对象共享同一个 Point：

```go
type ColoredPoint struct {
    *Point
    Color color.RGBA
}
q.Point = p.Point  // 共享同一个 Point
p.ScaleBy(2)       // q 也会受影响
```

### 5. 多级嵌入与二义性

一个 struct 可以嵌入多个匿名字段，方法查找按层级递进：先找直接定义的方法，再找第一层嵌入的，再向下递归。同一层有两个同名方法时编译器报错（二义性）。

匿名 struct 也可以嵌入，常见于将锁和数据组合：

```go
var cache = struct {
    sync.Mutex
    mapping map[string]string
}{mapping: make(map[string]string)}

cache.Lock()
v := cache.mapping[key]
cache.Unlock()
```

### 6. 方法值

选择器 `p.Distance` 不立即调用，而是返回一个绑定了接收器 `p` 的函数值（方法值），之后调用时只需传方法参数：

```go
distanceFromP := p.Distance   // 绑定了 p 的函数值
distanceFromP(q)              // 相当于 p.Distance(q)

// 实用场景：传给需要函数值的 API
time.AfterFunc(10*time.Second, r.Launch) // 比 func() { r.Launch() } 更简洁
```

`ch6/ch6-04.md`

### 7. 方法表达式

`T.f` 或 `(*T).f` 返回一个普通函数，该函数将接收器作为第一个参数：

```go
distance := Point.Distance         // 类型：func(Point, Point) float64
distance(p, q)                     // 等价于 p.Distance(q)

scale := (*Point).ScaleBy          // 类型：func(*Point, float64)
scale(&p, 2)                       // 等价于 p.ScaleBy(2)
```

适合在运行时根据条件选择调用同一类型的不同方法：

```go
var op func(p, q Point) Point
if add { op = Point.Add } else { op = Point.Sub }
path[i] = op(path[i], offset)
```

`ch6/ch6-04.md`

### 8. String() 方法与 fmt

为类型定义 `String() string` 方法后，`fmt` 包会自动调用它来打印该类型的值（依赖接口机制，第7章详述）。

注意接收器类型影响 fmt 是否能自动调用：

```go
// *IntSet 有 String() 方法
fmt.Println(&x)         // "{1 9 42 144}"  — 自动调用
fmt.Println(x.String()) // "{1 9 42 144}"  — 编译器插入 &
fmt.Println(x)          // "{[4398046511618 0 65536]}" — IntSet 无 String，打印原始值
```

如果结构体本身（非指针）也需要友好打印，应将 `String()` 定义在值接收器上。`ch6/ch6-05.md`

### 9. 封装

Go 封装的唯一手段是**首字母大小写**：大写导出，小写包内可见。封装的最小单元是 **package**，不是类型——同包的所有代码都能访问小写字段。

封装三大优点：
1. 调用方只需关注少量接口，降低理解成本。
2. 隐藏实现细节，作者可以自由改变内部实现（如 bytes.Buffer 的预分配优化）。
3. 防止外部随意修改内部状态，保证类型不变性（如 Counter 只能 Increment/Reset）。

getter 方法命名约定：省略 `Get` 前缀，直接用字段语义命名（如 `Flags()` 而非 `GetFlags()`）；setter 用 `SetXxx`。`ch6/ch6-06.md`

**何时不封装**：当类型本质就是某种基础类型的别名，且不会增加额外字段时，可以直接暴露（如 `geometry.Path` 是 `[]Point`，直接导出 slice 让调用方可以 range 遍历）。

## 重点关注点

- 接收器名用类型首字母，保持一致，不用 this/self。
- 指针接收器 vs 值接收器：修改状态或对象大时用指针，否则可用值。
- 一个类型有指针接收器方法，则全部方法应统一用指针接收器。
- 嵌入是组合（has-a），不是继承（is-a），类型不能互相替换。
- 方法查找按嵌入层级递进，同层二义性报错。
- 封装粒度是 package，同包代码均可访问小写字段。
- 为类型定义 `String()` 可以控制 fmt 的打印行为。
