## 知识点

### 1. reflect.Type 和 reflect.Value

`reflect.TypeOf(x)` 返回动态类型，`reflect.ValueOf(x)` 返回动态值。`Value` 可以继续查看 Kind、字段、元素、方法等。

### 2. Type 和 Kind

Type 表示具体类型，Kind 表示底层分类。例如 `type MyInt int` 的 Type 是 `MyInt`，Kind 是 `int`。

### 3. 可寻址和可设置

不是所有 `reflect.Value` 都能修改。要修改值，通常需要传指针，并通过 `Elem()` 取得可设置的值。

```go
v := reflect.ValueOf(&x).Elem()
v.SetInt(10)
```

### 4. struct tag

结构体字段标签通过反射读取，常见于 JSON、数据库映射、表单绑定。

```go
field.Tag.Get("json")
```

### 5. 反射成本

反射代码编译期检查少，运行时错误多，性能也通常弱于直接代码。能用接口、泛型、普通函数解决时，不要优先用反射。

## 重点关注点

- `Type` 是具体类型，`Kind` 是类别。
- 修改值必须满足可寻址、可设置。
- 反射经常和指针、接口动态类型一起出现。
- struct tag 本身只是字符串，语义由库解释。
- 反射是最后手段，不是日常抽象工具。
