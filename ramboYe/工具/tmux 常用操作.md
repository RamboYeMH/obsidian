# tmux 常用操作

> tmux 是终端复用器，核心作用：让程序跑在「脱离终端窗口」的会话里。关终端、断 SSH 程序都继续活着，回来重新接回即可。
> ⚠️ tmux 扛得住「终端断开」，扛不住「机器关机/重启/断电」——关机后会话彻底消失，无法恢复。

## 指挥键（前缀键）

默认前缀键是 **`Ctrl+b`**：**先按下松开，再按第二个键**，不是同时按。
下文写「`Ctrl+b` → `s`」即表示先按 `Ctrl+b`，松开，再按 `s`。

## 一、会话 session（大单位 / 独立工作空间）

| 操作           | 命令                                         |
| ------------ | ------------------------------------------ |
| 新建会话（终端外）    | `tmux new -s 名字`（如 `tmux new -s cc`）       |
| 新建会话（tmux 内） | `Ctrl+b` → `:` 再输入 `new-session -s 名字`     |
| ⭐ 列出并切换会话    | `Ctrl+b` → `s`（弹列表，方向键选，回车进）               |
| 切上一个 / 下一个会话 | `Ctrl+b` → `(` / `Ctrl+b` → `)`            |
| 脱离当前会话（不关闭）  | `Ctrl+b` → `d`（detach）                     |
| 查看有哪些会话（终端外） | `tmux ls`                                  |
| 接回某个会话（终端外）  | `tmux attach -t 名字`（如 `tmux attach -t cc`） |

> **脱离（detach）≠ 关闭**。脱离只是把「显示器」拔掉，会话和里面的进程仍在后台跑。

## 二、窗口 window（同一会话里的「标签页」）

| 操作          | 命令                              |
| ----------- | ------------------------------- |
| 新建窗口        | `Ctrl+b` → `c`（create）          |
| 下一个 / 上一个窗口 | `Ctrl+b` → `n` / `Ctrl+b` → `p` |
| 跳到第 N 个窗口   | `Ctrl+b` → `0`~`9`              |
| 列出所有窗口选择    | `Ctrl+b` → `w`                  |

## 三、面板 pane（一个窗口里分屏）

| 操作 | 命令 |
|------|------|
| 左右分屏 | `Ctrl+b` → `%` |
| 上下分屏 | `Ctrl+b` → `"` |
| 面板间移动光标 | `Ctrl+b` → 方向键 |

## 典型用法：挂着跑 Claude Code

```bash
tmux new -s cc        # 1. 新建会话
claude                # 2. 在里面启动 Claude Code，设好 /loop
# Ctrl+b d            # 3. 脱离，回到原终端；cc 会话和 claude 继续后台跑
# ...关终端 / 断 SSH 都没事...
tmux attach -t cc     # 4. 回来重新接回，状态原样还在
```

## 适用边界

| 场景 | tmux 是否能保住进程 |
|------|---------------------|
| 关终端窗口、断 SSH、网络掉线 | ✅ 能 |
| 机器关机 / 重启 / 断电 / 崩溃 | ❌ 不能，会话彻底消失 |

- **远程服务器**长时间挂任务 → tmux 最合适（本地终端随便关）。
- **本地机器关机后还要跑** → tmux 救不了，改用云端 `/schedule` 或本地 `cron` / `systemd` + `claude -p`。
- 插件 `tmux-resurrect` 只能恢复窗口布局，**恢复不了进程状态**（claude 上下文、`/loop` 进度都不回来），对长跑任务基本无用。
